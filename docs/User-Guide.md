## 快速开始

### 1. 安装

从 [GitHub Releases](https://github.com/PM-Shawn/Abu-Cowork/releases) 下载对应平台的安装包。首次安装遇到安全提示，请参考 [安装指南](Installation-Guide.md)。

### 2. 配置模型

1. 打开 Abu，点击左下角 **设置图标**
2. 进入 **「自定义模型」** 页面
3. 选择 API 服务商（如 Anthropic、DeepSeek、通义千问等）
4. 填入 API Key
5. 选择要使用的模型

### 3. 开始对话

回到主界面，在输入框中用自然语言描述你的需求，阿布会自动规划任务并执行。

## 对话与 Agent

Abu 的核心是 **Agent 自主执行模式**，不是简单的问答聊天。

### 工作流程

1. **理解需求** — 阿布分析你的指令，理解要完成的任务
2. **制定计划** — 自主拆解任务步骤
3. **调用工具** — 根据需要读写文件、执行命令、搜索信息
4. **迭代完善** — 根据执行结果调整策略，直到任务完成
5. **汇报结果** — 告诉你完成了什么、生成了哪些文件

### 权限确认

阿布在执行敏感操作时会请求你的确认：

- **命令执行** — 首次运行 shell 命令前会询问是否允许
- **文件写入** — 创建或修改文件前会展示变更内容
- **路径访问** — 访问敏感目录时需要你授权

你可以选择 **允许** 或 **拒绝**，也可以设置 **始终允许** 特定操作。

### 对话管理

- 左侧边栏显示所有历史对话
- 点击 **「新对话」** 开始新的任务
- 支持对话搜索和删除

## MCP 工具协议

MCP（Model Context Protocol）让阿布连接各种外部服务和工具。

### 什么是 MCP？

MCP 是一个开放协议，让 AI 助手通过标准化接口调用外部工具。比如：

- 连接 **数据库**，查询和分析数据
- 对接 **GitHub**，管理仓库和 Issue
- 调用 **搜索引擎**，获取实时信息
- 连接 **Slack/飞书**，发送消息

### 添加 MCP 服务器

1. 打开 **工具箱** → **MCP 工具** 标签页
2. 点击 **「添加 MCP 服务器」**
3. 选择连接方式：
   - **Stdio** — 本地命令行工具（最常用）
   - **HTTP** — 远程 HTTP 服务
4. 填入服务器配置（命令、参数、环境变量等）
5. 点击 **「连接」**

### MCP 配置示例

**连接文件系统服务器：**
```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"],
  "env": {}
}
```

**连接 GitHub 服务器：**
```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-github"],
  "env": {
    "GITHUB_PERSONAL_ACCESS_TOKEN": "your-token-here"
  }
}
```

### MCP 发现

你也可以在对话中让阿布帮你搜索和安装 MCP 服务器：

```
帮我搜索有没有连接 Notion 的 MCP 服务器
```

## 定时任务

让阿布按计划自动执行重复性工作。

### 创建定时任务

**方法一：对话创建**

直接告诉阿布你想定时做什么：

```
每天早上 9 点帮我搜索最新的 AI 新闻，写一份日报发到桌面
```

```
每周一早上 10 点，帮我整理上周的会议笔记，生成周报
```

**方法二：定时任务面板**

1. 点击侧边栏的 **定时任务** 图标
2. 点击 **「创建任务」**
3. 设置频率（每小时 / 每天 / 每周 / 自定义）
4. 填写任务描述
5. 保存

### 频率选项

| 频率 | 说明 |
|------|------|
| 每小时 | 每小时执行一次 |
| 每天 | 每天指定时间执行 |
| 每周 | 每周指定日期和时间执行 |
| 自定义 | 自定义执行间隔 |

### 管理定时任务

- **暂停/恢复** — 临时暂停某个任务
- **编辑** — 修改任务描述或频率
- **删除** — 永久删除任务
- **查看历史** — 查看每次执行的结果

### 注意事项

- 定时任务在 **无人值守** 模式下运行，不会弹出确认对话框
- 已授权的路径和命令会自动放行，未授权的敏感操作会自动跳过
- 任务完成后会发送 **桌面通知**
- Abu 需要保持运行才能执行定时任务

## 触发器

触发器让阿布**自动响应外部事件**——收到 Webhook、文件变化、定时周期、IM 消息时，自动执行你预设的任务。

### 创建触发器

1. 点击侧边栏的 **触发器** 图标
2. 点击 **「新建触发器」**（也可从模板创建）
3. 填写触发器名称和描述
4. 选择事件来源、过滤条件、执行动作
5. 保存

### 事件来源

触发器支持 4 种事件来源：

| 来源类型 | 说明 | 配置项 |
|---------|------|--------|
| **HTTP Webhook** | 接收外部系统的 POST 请求 | 自动生成唯一端点 URL，复制配置到外部系统即可 |
| **文件监控** | 监听文件/目录变化 | 监控路径、事件类型（创建/修改/删除）、文件匹配模式（如 `*.log`） |
| **定时器** | 按固定间隔执行 | 执行间隔（秒），最低 10 秒 |
| **IM 消息** | 监听 IM 平台消息 | 平台选择、App ID/Secret、监听范围（全部/仅@/仅私聊） |

### 过滤条件

控制什么事件才触发执行：

| 过滤类型 | 说明 |
|---------|------|
| **始终执行** | 每次事件都执行 |
| **关键词匹配** | 事件内容包含指定关键词才执行（逗号分隔多个关键词） |
| **正则匹配** | 事件内容匹配正则表达式才执行 |

### 防抖去重

开启后，相同事件在指定时间窗口内只会触发一次。适合高频事件（如日志写入、重复告警）。默认窗口 300 秒。

### 静默时段

可设置时间范围（如 22:00 ~ 08:00），在此期间不执行触发器，避免半夜打扰。支持跨零点设置。

### 执行动作

触发器被激活后，阿布会执行你写的 **提示词指令**。你可以在指令中使用 `$EVENT_DATA` 引用事件原始数据。

还可以：
- **绑定技能** — 选择一个已有的 Skill，触发时自动调用
- **指定工作目录** — 让阿布在特定目录下执行

### 输出配置

触发器执行完成后，可以将结果推送到外部：

| 输出目标 | 说明 |
|---------|------|
| **Webhook** | 将结果推送到指定 URL（支持自定义 Header 和提取模板） |
| **回复来源** | 直接回复触发事件的 IM 消息 |

**提取模板变量**：`$TRIGGER_NAME`、`$EVENT_SUMMARY`、`$AI_RESPONSE`、`$RUN_TIME`、`$TIMESTAMP`、`$EVENT_DATA`

### 内置模板

Abu 提供 3 个快速上手模板：

| 模板 | 说明 |
|------|------|
| **告警消息处理** | 预设关键词 `error, alert, warning, P0, P1, critical`，自动分析告警严重性并给出建议 |
| **日志文件监控** | 监控日志文件变化，自动分析新条目、识别异常 |
| **定期健康检查** | 定时检查磁盘、内存、关键进程，生成报告 |

### 运行历史

每个触发器会记录最近的执行历史，在侧边栏 **「触发器」** 面板中可以查看：

- 执行状态（运行中 / 已完成 / 出错 / 被过滤 / 被去重）
- 执行时间
- 关联的对话记录
- 成功率统计

## IM 频道

IM 频道让外部用户通过即时通讯平台（飞书、钉钉等）直接与阿布对话，无需打开 Abu 桌面端。

### 支持的平台

| 平台 | 说明 |
|------|------|
| **飞书** | 支持 WebSocket 长连接，无需公网 IP |
| **钉钉** | 通过 Webhook 回调 |
| **企业微信** | 通过 Webhook 回调 |
| **Slack** | 通过 Events API |
| **D-Chat** | 通过 Webhook 回调 |

### 快速配置（以飞书为例）

#### 第一步：在飞书开放平台创建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)，创建一个企业自建应用
2. 获取 **App ID** 和 **App Secret**
3. 在「事件订阅」中添加 `im.message.receive_v1` 事件
4. 开启「机器人」能力

#### 第二步：在 Abu 中添加频道

1. 打开 **设置** → **IM 频道**
2. 点击 **「添加频道」**
3. 填写频道名称（如「研发群机器人」）
4. 选择平台为 **飞书**
5. 填入 App ID 和 App Secret
6. 选择能力等级
7. 保存

#### 第三步：配置回调地址

保存后会自动生成一个 **回调地址**（Webhook URL），将它复制并粘贴到飞书开放平台的「事件订阅」请求地址中。

> 飞书平台也支持 WebSocket 模式，Abu 会自动建立长连接，此时无需配置回调地址和公网 IP。

#### 第四步：测试

在飞书中给机器人发消息或在群里 @机器人，Abu 会自动回复。

### 能力等级

控制 IM 用户通过阿布能做什么：

| 等级 | 权限范围 | 适用场景 |
|------|---------|---------|
| **仅对话** | 只能聊天，不能操作任何文件 | 面向所有人的问答机器人 |
| **只读** | 可以查看文件内容，不能修改 | 需要查阅代码/文档的场景 |
| **标准** | 可以读写已授权目录下的文件 | 日常开发协作（推荐） |
| **完全控制** | 拥有完整权限，包括执行命令 | 仅限信任的管理员，需配合白名单使用 |

### 白名单

- **留空**：允许所有人使用。如果能力等级设为「完全控制」，非白名单用户会自动降为「标准」权限
- **填入用户 ID**：只有白名单内的用户才能使用该频道

### 会话管理

IM 频道会自动管理对话的生命周期：

- **会话超时**：用户一段时间没有发消息后，下次发消息会自动开启新对话（默认 30 分钟）
- **最大轮次**：单个对话达到轮次上限后自动开新对话（默认 50 轮）
- **恢复上下文**：用户发送「继续上次」可以恢复上一个已结束的对话
- **会话隔离**：群聊中不同用户的对话互相独立，不会串台

### 在 Abu 中查看 IM 对话

IM 用户发来的消息会在 Abu 侧边栏的 **「最近」** 列表中显示。点击即可查看完整对话内容。对话顶部的信息栏显示来源平台、对话轮次等信息，点击右侧 **⋯** 菜单可查看详情或结束会话。

### 注意事项

- Abu 需要保持运行才能接收和回复 IM 消息
- 飞书 WebSocket 模式不需要公网 IP；其他平台的 Webhook 模式需要 Abu 所在机器能被外网访问（或使用内网穿透工具）
- IM 模式下，阿布不会弹出桌面确认对话框，涉及需要确认的操作会自动跳过
- 建议为不同团队/用途创建不同的频道，便于权限管理

## 浏览器操控

通过 Abu Browser Bridge 和 Chrome 扩展，阿布可以操控浏览器完成网页任务。

### 安装配置

1. **安装 Chrome 扩展**
   - 打开 Chrome，进入 `chrome://extensions/`
   - 开启「开发者模式」
   - 点击「加载已解压的扩展程序」，选择 `abu-chrome-extension` 目录

2. **启动 Bridge 服务**
   - Abu 会自动管理 Browser Bridge 的启动和连接

### 功能

| 功能 | 说明 |
|------|------|
| **页面快照** | 获取当前页面的结构化信息（元素、文本、状态） |
| **元素点击** | 点击页面上的按钮、链接等元素 |
| **表单填写** | 自动填写输入框、下拉框等表单控件 |
| **页面导航** | 打开网址、前进后退、切换标签页 |
| **截图** | 对当前页面截图 |
| **等待条件** | 等待元素出现/消失、URL 变化等 |
| **执行脚本** | 在页面中执行 JavaScript 代码 |

### 使用示例

```
打开百度，搜索"Abu AI 助手"，把搜索结果前 5 条整理成表格
```

```
打开我的 GitHub 仓库，检查有没有新的 Issue，把内容整理发给我
```

## AI 服务配置

Abu 支持多种 LLM 服务商，并提供 **对话**、**联网搜索**、**图片生成** 三大 AI 能力。

打开 **设置** → **AI 服务** 可以看到当前配置状态。

### 支持的服务商

| 服务商 | 内置联网搜索 | 说明 |
|--------|:---:|------|
| **Anthropic** | ✅ | Claude 系列模型，推荐使用 |
| **火山引擎** | ✅ | 字节跳动云服务，豆包模型 |
| **阿里百炼** | ✅ | 阿里云大模型平台，通义千问等 |
| **智谱 AI** | ✅ | 清华技术团队，GLM 系列 |
| **Moonshot** | ✅ | Kimi 背后的模型 |
| **OpenAI** | — | GPT 系列模型 |
| **硅基流动** | — | 多模型聚合平台 |
| **DeepSeek** | — | 性价比高，支持推理模型 |
| **七牛云** | — | 多模型聚合，15+ 模型 |
| **OpenRouter** | — | 海外模型聚合路由 |
| **本地模型** | — | Ollama、LM Studio 等 |
| **自定义 API** | — | 任意 OpenAI/Anthropic 兼容接口 |

> **✅** 表示服务商内置支持联网搜索，选择后可直接使用，无需额外配置。
> **—** 表示服务商不内置此能力，但你可以通过「自定义配置」单独接入。

### 模型配置步骤

1. 打开 **设置** → **AI 服务**
2. 选择 **服务商**（如 Anthropic、DeepSeek、阿里百炼等）
3. 填写 **API Key**
4. 选择 **模型**（每个服务商提供不同的可选模型）
5. （可选）展开 **高级选项** 调整温度参数

### 自定义 API 配置

选择「自定义 API」服务商时，需额外填写：

- **API 地址** — 服务的 Base URL
- **模型名称** — 模型 ID
- **API 协议** — `OpenAI Compatible` 或 `Anthropic`
- **API Key**

### 本地模型配置

使用 Ollama 等本地模型：

1. 服务商选择 **「本地模型」** 或 **「自定义 API」**
2. API 地址填入 `http://localhost:11434/v1`（Ollama 默认）
3. API Key 填任意值（如 `ollama`）
4. 模型名称填入本地模型名（如 `llama3`）

### 高级参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **温度 (Temperature)** | 控制回复的随机性，越低越确定 | 0.7 |
| **扩展思维 (Extended Thinking)** | 让模型在回答前进行深度推理（支持的模型） | 关闭 |
| **思维预算** | 扩展思维的 Token 预算 | 10000 |

## 沙箱与安全

Abu 内置多层安全保护机制，确保 AI 操作不会损害你的系统。

### OS 级沙箱

| 平台 | 技术 | 效果 |
|------|------|------|
| macOS | Seatbelt (sandbox-exec) | 限制 shell 命令的文件访问范围 |
| Windows | PowerShell ConstrainedLanguage | 限制脚本执行能力 |

### 网络隔离

- **域名白名单** — 只允许访问白名单内的域名
- **私有网络控制** — 可控制是否允许访问本地网络（127.0.0.1、192.168.* 等）
- **代理机制** — 通过本地代理实现网络流量管控

### 路径保护

阿布不会在未经许可的情况下访问：

- 系统目录（`/System`、`/usr`、`C:\Windows` 等）
- 其他用户目录
- 敏感配置文件（SSH 密钥、浏览器数据等）

### 命令安全

- 危险命令（如 `rm -rf /`）会被自动拦截
- 首次执行的命令需要用户确认
- 已授权的命令可以自动放行

### 配置沙箱

1. 打开 **设置** → **安全沙箱**
2. 开启/关闭 **OS 沙箱**
3. 开启/关闭 **网络隔离**
4. 管理 **域名白名单**

## 常见问题

### Q: Abu 会上传我的数据吗？

不会。Abu 是本地优先的应用，你的文件和数据只在本地处理。唯一的网络通信是与 LLM API 服务商之间的对话请求。

### Q: API Key 存在哪里？

API Key 加密存储在本地应用数据目录中，不会上传到任何服务器。

### Q: 可以同时使用多个模型吗？

目前一次只能激活一个模型配置。你可以在设置中随时切换不同的服务商和模型。

### Q: 定时任务不执行怎么办？

- 确保 Abu 应用正在运行（不能关闭应用）
- 检查任务是否处于「暂停」状态
- 查看任务执行历史中的错误信息

### Q: 如何创建自定义 Skill？

在对话中说「帮我创建一个新技能」，阿布会引导你完成整个流程。技能以 Markdown 文件形式存储在 `builtin-skills/` 目录下，你也可以直接编辑。

### Q: MCP 服务器连接失败？

- 确认 MCP 服务器的命令和参数正确
- 检查是否安装了必要的运行时（如 Node.js、Python）
- 查看环境变量是否正确配置
- 如果是 HTTP 类型，确认 URL 可访问

### Q: 触发器和定时任务有什么区别？

定时任务只支持按时间周期执行。触发器更灵活——除了定时，还能响应 Webhook 请求、文件变化、IM 消息等外部事件，并支持过滤、去重、静默时段、输出推送等高级功能。

### Q: IM 频道需要公网 IP 吗？

飞书支持 WebSocket 长连接模式，Abu 主动连接飞书服务器，**不需要公网 IP**。其他平台使用 Webhook 回调模式，需要 Abu 所在机器能被外网访问（可以使用 ngrok 等内网穿透工具）。

### Q: IM 用户能执行危险操作吗？

取决于频道的「能力等级」设置。建议使用「标准」等级（默认），阿布只能读写已授权的目录。「完全控制」等级必须配合白名单使用，只有白名单内的用户才能获得完整权限。

### Q: 如何使用浏览器操控功能？

1. 安装 Abu Chrome 扩展
2. 确保 Browser Bridge 服务已连接（工具箱中可查看状态）
3. 在对话中描述你想在浏览器中完成的操作

### Q: 支持哪些语言？

Abu 界面支持 **简体中文** 和 **English**，可在设置中切换。也可以设为「跟随系统」自动检测。
